JavaScript/API/Geolocation

Die Geolocation API kann verwendet werden, um den Standort des Clienten zu ermitteln, was vor allem für mobile Anwendungen interessant ist. Bestimmt werden kann der Standort entweder via Internetanbindung oder GSM/GPS. Mit den Daten zum Standort können die Genauigkeit der Angaben und Bewegungen aufgezeichnet werden.

Mittels Google Gears oder einer IP-Datenbank lässt sich der Standort des Clienten ebenfalls ermitteln, ohne dass er vom Browser gefragt wird, ob die Standortdaten übermittelt werden dürfen. Hierdurch lässt sich eine mögliche Verunsicherung des Clienten vermeiden. Dies ist vor allem deshalb sinnvoll, da nur die wenigsten und neusten Browserversionen dieses Feature unterstützen. Ohne Gears oder die Nutzung einer IP-Datenbank würde also bei den meisten Clienten an Stelle der gewünschten Ausgabe ein Fehler auftreten. Allerdings wird Gears nicht mehr weiterentwickelt und von neueren Browsern auch nicht mehr unterstützt, da das Projekt durch HTML5 nicht mehr benötigt wird.

if (navigator.geolocation) { 
     /* code */ 
}

[Bearbeiten] Abfragen der Position 

navigator.geolocation.getCurrentPosition( /* position callback */ );

[Bearbeiten] Position verfolgen 

//Verfolgen beginnen
var watchID = navigator.geolocation.watchPosition( /* position callback */ );
 
//Verfolgen beenden
navigator.geolocation.clearWatch(watchID);

[Bearbeiten] Funktion für den Position Callback 

function positionCallback(position) {
     //wann die Position ermittelt wurde
     position.timestamp;
 
     position.coords.latitude; //dezimal Grad
     position.coords.longitude; //dezimal Grad
     position.coords.accuracy; //Meter
 
     position.coords.altitude; //Meter
     position.coords.altitudeAccuracy; //Meter
 
     position.coords.speed; //Meter pro Sek.
     position.coords.heading; //Grad von wahrem Norden
}

[Bearbeiten] Error Callback 

navigator.geolocation.getCurrentPosition(callback, errorCallback); //Für Positionsabfrage
navigator.geolocation.watchPosition(callback, errorCallback);  //Für Verfolgung
 
function errorCallback(error) {
     var err = error.code;
 
     switch (err) {
          case error.PERMISSION_DENIED: break;
          case error.POSITION_UNAVAILABLE: break;
          case error.TIMEOUT: break;
          case error.UNKNOWN_ERROR: break;
     }
 
     window.console.log(error.message);
}

[Bearbeiten] Weitere Optionen

bool enableHighAccuracy
Nutzen der GPS-Ortung aktivieren/deaktivieren. Sie bringt eine bessere Genauigkeit, hat aber höheren Akkuverbrauch zur Folge.
long timeout //In Millisekunden
Zeit, die man bereit ist, auf ein Callback zu warten.
long maximumAge //In Millisekunden
Wenn alte Geodaten ausreichend sind, wird auf diese zurückgreifen.

Von diesen Optionen können ein, zwei oder alle drei wie folgt eingebunden werden, wobei die erste Zeile alle drei Parameter zeigt, die zweite nur einen: 

navigator.geolocation.getCurrentPosition(callback, errorCallback, {enableHighAccuracy:true, timeout:5000, maximumAge:60000});
navigator.geolocation.watchPosition(callback, errorCallback, {enableHighAccuracy:true});
