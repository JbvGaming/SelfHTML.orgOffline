[Bearbeiten] das rev Attribut

Es gilt als deprecated und existiert nicht mehr in HTML5

Theoretisch gilt:

  • Dokument A enthält:

<link rel="next" href="B">

  • Dokument B könnte enthalten:

<link rev="next" href="A">

  • Dokument B sollte aber besser enthälten:

<link rel="prev" href="A">

--Beat 09:49, 9. Mai 2010 (CEST)

[Bearbeiten] type

Das type-Attribut wird eigentlich nicht benötigt, denn üblicherweise senden die Webserver den MIME-Type einer Ressource (z.B. CSS, Bilder, JavaScript) in deren HTTP-Header-Daten mit. Wenn diese Angabe fehlt, darf der Browser sie aus diesem Attribut des verlinkenden Elements nehmen.

Das kommt mir arg hypothetisch vor. Zum einen gibt es auf Linuxservern üblicherweise die Datei /etc/mime.types, die gängige und weniger gängige Typen abdeckt, zum anderen wird bei Apache und Co. ein vorgegebener Typ verwendet, wenn kein passender Eintrag gefunden wurde. – Bmn 20:13, 17. Feb. 2011 (CET)

Zumindest für das Encoding gibt es eine Reihenfolge, in der definiert ist, dass das charset-Attribut in Verweisen als eines der letzten Hinweise genommen werden darf. Ich schlussfolgerte daraufhin fälschlicherweise eine Analogie beim type-Attribut. Allerdings ist dieses für den Browser nur eine hinweisende Vorwarnung, welchen Typ er zu erwarten hat, damit er sich das Laden gleich ganz sparen kann, wenn er den Typ nicht unterstützt.
This attribute gives an advisory hint as to the content type of the content available at the link target address. It allows user agents to opt to use a fallback mechanism rather than fetch the content if they are advised that they will get content in a content type they do not support.
Ich werde das mal richtigstellen. --dedlfix 11:34, 18. Feb. 2011 (CET)
