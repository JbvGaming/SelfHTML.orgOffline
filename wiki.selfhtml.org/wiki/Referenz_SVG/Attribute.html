[Bearbeiten] Kernattribute

SVG definiert die folgenden vier Kernattribute, die an jedem SVG-Element erlaubt sind. Keines der Atrribute ist erforderlich.

Attribut Wert Bedeutung
id [ID] ordnet dem Element einen eindeutigen Bezeichner „ID“ zu
xml:base #CDATA gibt die Basis-URI an
xml:id [ID] ordnet dem Element einen eindeutigen Bezeichner „ID“ zu
xml:space preserve|default bestimmt, wie Leerzeichen im Quelltext interpretiert werden

[Bearbeiten] XLink-Attribute

Zur Auszeichnung von Hyperlinks bedient sich SVG des XLink-Standards. Keines der Attribute ist erforderlich. Die folgenden Attribute dieses Standards werden dabei an referenzierenden Elementen verwendet:

Attribut Wert Bedeutung
xlink:actuate onLoad bietet Dokumentation für XLink-verarbeitende Programme (nicht bei den Elementen linearGradient, radialGradient, textPath und tref)
xlink:arcrole [IRI] verweist auf eine Resource, die die eigentlich verlinkte Resource näher bestimmt
xlink:role [IRI] verweist auf eine Resource, die die eigentlich verlinkte Resource näher bestimmt
xlink:show new|replace|embed|other|none bietet Dokumentation für XLink-verarbeitende Programme (wird im Zweifel von target überschrieben; nicht bei den Elementen linearGradient, radialGradient, textPath und tref)
xlink:title #CDATA bestimmt einen Beschreibungstext für die verlinkte Resource
xlink:type simple identifiziert den Link als Typ „simple“

[Bearbeiten] Grafische Ereignis-Attribute

Grafische und Container-Elemente können der Ausgangspunkt für grafische Ereignisse sein, die z.B. Javascript-Aktionen starten. Ereignisse funktionieren hierbei auf eine vergleichbare Art wie in HTML. Keines der Attribute ist erforderlich. Die folgenden Attribute erlauben die Zuordnung eines Ereignisses zu einem Element:

Attribut Wert Bedeutung
onactivate #CDATA wird aktiviert, wenn das Element aktiviert wird, z.B. durch Mausklick
onclick #CDATA wird aktiviert, wenn das Element geklickt wird
onfocusin #CDATA wird aktiviert, wenn das Element den Fokus erhält
onfocusout #CDATA wird aktiviert, wenn das Element den Fokus verliert
onload #CDATA wird aktiviert, wenn das Element einschließlich Kindelementen geladen wird
onmousedown #CDATA wird aktiviert, wenn eine Maustaste über dem Element gedrückt wird
onmousemove #CDATA wird aktiviert, wenn die Maus über dem Element bewegt wird
onmouseout #CDATA wird aktiviert, wenn die Maus das Element verläßt
onmouseover #CDATA wird aktiviert, wenn die Maus über das Element fährt
onmouseup #CDATA wird aktiviert, wenn eine Maustaste über dem Element losgelassen wird

[Bearbeiten] Animations-Takt-Attribute

Zur Animation verwendet SVG Teile des SMIL-Standards. Um die zeitliche Abfolge von Animationen zu definieren, werden folgende Attribute verwendet:

Attribut Wert Bedeutung
begin #CDATA definiert, wann das Element beginnen soll
dur #CLOCK_VALUE|media|indefinite spezifiziert die einfache Dauer
end #CDATA definiert einen Endwert, der die Animation beschränken kann
fill freeze|remove legt fest, ob nach Ablauf der Animation der Urzustand wiederhergestellt werden soll
max #CLOCK_VALUE|media legt die Maximaldauer der Animation fest
min #CLOCK_VALUE|media legt die Minimaldauer der Animation fest
repeatCount #NUMBER|indefinite bestimmt die Anzahl der Wiederholungen
repeatDur #CLOCK_VALUE|indefinite bestimmt die Gesamtdauer der Wiederholungen
restart always|whenNotActive|never legt fest, wie die Animation wiederholt werden soll
