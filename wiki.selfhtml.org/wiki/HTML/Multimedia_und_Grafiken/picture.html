Mithilfe des picture-Elements können Sie zukünftig Bilder auf Webseiten darstellen. Dabei können Sie alternative Bildquellen für verschiedene Viewports angeben.

  • 38
  • 25

Bisher konnten Sie die Größe des img-Elements mithilfe von CSS an die Größe des Viewports anpassen. Allerdings wurden auch für mobile Geräte hochauflösende Bilder eingebunden, die aber bei geringen Übertragungsraten nur langsam geladen wurden.


Beispiel: Einbinden von Bilddateien mit img
img { max-width: 100%; height: auto; }

[Bearbeiten] mehrere Bildquellen mit source

Das picture-Element ermöglicht nun die Angabe mehrerer Bildquellen, so dass nur das passende Bild geladen werden muss, was bei geringen Bandbreiten Übertragungszeit spart. Als Fallback wird am Ende ein img zur Verfügung gestellt.

  • 33
  • 38
  • 25
Beispiel: Einbinden von Bildateien mit picture
<!doctype html> <html> <head> <meta charset="utf-8"> </head> <body> <picture> <source media="(min-width: 1024px)" srcset="feuerwehr-1600.jpg"> <source media="(min-width: 480px)" srcset="feuerwehr-480.jpg"> <!---Fallback---> <img src="feuerwehr-480.jpg" srcset="feuerwehr-320.jpg" alt="Feuerwehrfest 2014"> </picture> </body> </html>

Folgende Attribute sind für source möglich:

  • srcset: durch Komma getrennte Liste der möglichen Medienquellen. Jeder String besteht aus:
    • URL:
    • Breite: positive Zahl gefolgt von einem w. (Defaultwert ist unendlich)
    • Pixeldichte: positive Zahl gefolgt von einem x. (Defaultwert ist 1x)
  • media: ermöglicht mit media queries der zu ladenden Medienquellen (nur im picture-Element) Breakpoints zu erstellen
  • type:: MIME-Typ der zu ladenden Medienquellen.
    (Falls der Medientyp nicht angegeben ist, wird er vom Server geladen und überprüft, ob der Browser das rendern kann. Falls nicht wird die nächste Medienquelle überprüft.
    Durch die Angabe des Medientyps wird nur das source-Element überprüft und, falls der Typ nicht verarbeitet werden kann, gleich die (erste)passende Quelle geladen.)
  • sizes: Größenangabe, Pixeldichte

[Bearbeiten] mehrere Bildquellen mit srcset

Mit dem Attribut srcset können Sie Bilder in verschiedenen Größen referenzieren, die dann entsprechend geladen werden.

Beispiel: Einbinden von Bildateien mit picture und scrset
<!doctype html> <html> <head> <meta charset="utf-8"> </head> <body> <picture> <source scrset="feuerwehr-1600.jpg, feuerwehr-1600hd.jpg 2x" media="(min-width: 1024px)"> <source scrset="feuerwehr-480.jpg, feuerwehr-480hd.jpg 2x" media="(min-width: 480px)"> <!---Fallback---> <img src="feuerwehr-480.jpg" scrset="feuerwehr-320.jpg, feuerwehr-320hd.jpg 2x" alt="Feuerwehrfest 2014"> </picture> </body> </html>

In diesem Beispiel werden im srcset mehrere URL angeboten. Browser ohne srcset-Support zeigen das mit img referenzierte Bild an, für Retina-Displays werden alternative hochauflösende Bilder zur Verfügung gestellt.

[Bearbeiten] mehrere Bildquellen mit sizes

Das Bild soll immer flächendeckend angezeigt werden. Je nach verfügbarem Viewport wird die passende Variante geladen.

Beispiel: Einbinden von Bildateien mit picture
<!doctype html> <html> <head> <meta charset="utf-8"> </head> <body> <img sizes="100vw" srcset="feuerwehr-mobil.jpg 480w, small-tablet.jpg 768w, tablet.jpg 968w, desktop.jpg 1024w" src="feuerwehr-480.jpg" alt="Feuerwehrfest 2014"> </body> </html>

Das Attribut:

  • sizes besteht aus einer durch Komma getrennten Liste, die die dargestellte Breite des Bildes beschreibt. Diese Information nutzt der Browser um zu entscheiden, welches der in srcset definierten Bildquellen geladen werden soll.
