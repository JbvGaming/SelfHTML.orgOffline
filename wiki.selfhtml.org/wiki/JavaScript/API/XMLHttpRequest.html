Das XMLHttpRequest-Objekt wird zum Austausch von Daten zwischen Nutzer und Server verwendet. Man kann damit ...

  • eine Webseite aktualisieren, ohne sie neu laden zu müssen
  • Daten vom Server anfordern, nachdem die Seite geladen ist
  • Daten vom Server empfangen, nachdem die Seite geladen ist
  • Daten zu einem Server im Hintergund senden
Beispiel: Erzeugung eines XMLHttpRequst-Objekts 
var ajax = null;
if(window.XMLHttpRequest){ //Google Chrome, Mozilla Firefox, Opera, Safari, IE 10
	ajax = new XMLHttpRequest();
}
else if(window.ActiveXObject){ // Internet Explorer 9 und niedriger
	try{
		ajax = new ActiveXObject("Msxml2.XMLHTTP.6.0");
	} catch(e){
		try{
			ajax = new ActiveXObject("Msxml2.XMLHTTP.3.0");
		}
		catch(e){}
	}
}
if(ajax==null)
	alert("Ihr Browser unterstützt kein Ajax!");

Im Code wird zuerst die Variable ajax mit null initialisiert, als nächstes wird getestet ob das XMLHttpRequest-Objekt (Ajax) existiert und ggf. erzeugt. Falls nicht (bei älteren Internet Explorer Versionen) wird das jeweilige ActiveX-Objekt abgefragt, und am Ende, für den Fall, dass das ajax-Objekt immer noch null ist, eine Fehlermeldung ausgegeben.

[Bearbeiten] Das Ajax-Objekt mit Daten füttern

Beispiel 
ajax.open("GET","daten.txt",true);
ajax.setRequestHeader("X-Test","test1");
ajax.setRequestHeader("X-Test","test2");

Als erstes "öffnen" wir das Ajax-Objekt und füttern es mit der verwendeten Methode, der URL und geben an ob synchron oder asynchron gearbeitet wird. Danach setzen wir noch einen Anfrage-Header. Wenn man nun 2 mal den gleichen Anfrage-Header (wie im Beispiel) angeben, kombiniert das Ajax-Objekt beide Werte.

[Bearbeiten] Die Verarbeitung bestimmen

Beispiel 
ajax.onreadystatechange = function(){
	if(this.readyState == 4){
		if(this.status == 200){
			alert(this.responseText);
		}
		else{
			alert(this.statusText);
		}
	}
}

Unsere Verarbeitung ist recht simpel, wenn nichts schiefgelaufen ist lassen wir uns den Inhalt der angeforderten Resource ausgeben, ansonsten den HTTP-Statustext. Nun ins Detail: Das Ajax-Objekt ruft intern die onreadystatechange-Funktion auf, sobald sich der Status (readyState) des Objektes/der Anfrage ändert. Am Interessantesten ist dabei der Wert 4. Dieser besagt, dass die Anfrage beendet wurde, dies bedeutet entweder, dass alles glatt verlaufen ist, oder das ein Fehler aufgetreten ist. Nun kontrollieren wir noch den Status der Antwort, 200 bedeutet, dass die Anfrage erfolgreich bearbeit und übertragen wurde. responseText beinhaltet den Inhalt der Resource.

[Bearbeiten] Die Anfrage senden

Beispiel 
ajax.send(null);

Die Anfrage wird gesendet. Wenn die verwendete Methode Daten versendet, wird der Parameter-Text übergeben, ansonsten null.

[Bearbeiten] Kompletter Code

Beispiel 
if(window.XMLHttpRequest){ //Google Chrome, Mozilla Firefox, Opera, Safari,...
	ajax = new XMLHttpRequest();
}
else if(window.ActiveXObject){ // Internet Explorer
	try{
		ajax = new ActiveXObject("Msxml2.XMLHTTP.6.0");
	} catch(e){
		try{
			ajax = new ActiveXObject("Msxml2.XMLHTTP.3.0");
		}
		catch(e){}
	}
}
if(ajax!=null){
	ajax.open("GET","daten.txt",true);
	ajax.setRequestHeader("X-Test","test1");
	ajax.setRequestHeader("X-Test","test2");
	ajax.onreadystatechange = function(){
		if(this.readyState == 4){
			if(this.status == 200){
				alert(this.responseText);
			}
			else{
				alert(this.statusText);
			}
		}
	}
	ajax.send(null);
}
else{
	alert("Ihr Browser unterstützt kein Ajax!");
}

[Bearbeiten] Beispiel: Benutzernamen-Prüfung

Hier wird nun das bereits erwähnte Beispiel mit Code hinterlegt.

Was wird gebraucht?

  • Eine serverseitige Resource zur Überprüfung des gewünschten Benutzernamens
  • Eine HTML-Seite mit Formular
  • Das eigentliche JavaScript

Desweiteren wird davon ausgegangen, das sich alle Dateien im gleichen Verzeichnis auf dem Webserver befinden.

[Bearbeiten] serverseitige Resource

Für das Beispiel wird PHP verwendet, getauft wird die Datei auf "usernamecheck.php".

Beispiel 
<?php
	if(!isset($_POST["username"]))
		die("{valid:false,message:false}");
 
	$usernames = Array("admin","gast","paul");
	$validatePattern = "/^[a-z0-9]{4,20}$/";
 
	if(in_array($_POST["username"]))
		die("{valid:false,message:'Der Benutzername wird bereits verwendet!'}");
	if(!preg_match($validatePattern,$_POST["username"]))
		die("{valid:false,message:'Der Benutzername muss aus kleinen Buchstaben(a-z) und/oder Ziffern(0-9) bestehen und 4-20 Zeichen lang sein'}");
	echo "{valid:true,message:false}";
?>

Das Script erwartet den Benutzernamen via POST-Methode und unter dem Namen username. Zurück gibt es einen JSON-Text, welches ein Objekt mit den Eigenschaften valid und message enthält.

[Bearbeiten] HTML-Seite

Beispiel 
<!DOCTYPE html>
<html>
	<head>
		<title>AJAX-Benutzerüberprüfung</title>
		<script type="text/javascript" src="check.js"></script>
	</head>
	<body>
		Benutzername: <input type="text" id="username"><button onclick="check();">check</button>
	</body>
</html>

[Bearbeiten] JavaScript-Datei

Dateiname: check.js

Beispiel 
function check(){
	if(window.XMLHttpRequest){ //Google Chrome, Mozilla Firefox, Opera, Safari,...
		ajax = new XMLHttpRequest();
	}
	else if(window.ActiveXObject){ // Internet Explorer
		try{
			ajax = new ActiveXObject("Msxml2.XMLHTTP.6.0");
		} catch(e){
			try{
				ajax = new ActiveXObject("Msxml2.XMLHTTP.3.0");
			}
			catch(e){}
		}
	}
	if(ajax!=null){
		ajax.open("POST","usernamecheck.php",true);
		ajax.onreadystatechange = function(){
			if(this.readyState == 4){
				if(this.status == 200){
					var obj = null;
					eval("obj = "+this.responseText);
					if(obj.valid){
						alert("Der Benutzername kann verwendet werden!");
					}
					else{
						if(obj.message){
							alert(obj.message);
						}
						else{
							alert("Ein unbekannter Fehler ist aufgetreten!");
						}
					}
				}
				else{
					alert("Es ist ein Fehler aufgetreten! ("+this.statusText+")");
				}
			}
		}
		ajax.send("username="+encodeURIComponent(document.getElementById("username").value));
	}
	else{
		alert("Ihr Browser unterstützt kein Ajax!");
	}
}

[Bearbeiten] Methoden

[Bearbeiten] XMLHttpRequest.open 

void function open(string method, string url[, boolean asynch = true[, string user = null[, string password = null]]])

Setzt die Anfrage-Methode, Anfrage-URL, ob die Anfrage asynchron abgearbeitet wird, Anfrage-Benutzername und Anfrage-Benutzerpasswort.

Ausnahmefehler: SyntaxError, wenn:

  • method ist keine gültige HTTP-Methode
  • url kann nicht aufgelöst werden
  • url beinhaltet das "user:password"-Format

SecurityError, wenn:

  • method ist eine der HTTP-Methoden CONNECT, TRACE oder TRACK

InvalidAccessError, wenn:

  • (SPÄTER)


[Bearbeiten] XMLHttpRequest.setRequestHeader 

void function setRequestHeader(string header, string value)

XMLHttpRequest.setRequestHeader setzt bestimmte Eigenschaften für die Resourcen-Anfrage. Wenn der gleiche Header mehrmals gesetzt wird, werden die Werte kombiniert.

Parameter:

  • header: Erwartet einen validen HTTP-Header-Feldnamen
  • value: Erwartet einen validen Wert für das Header-Feld

Rückgabewert: keine Rückgabe

Ausnahmefehler:

  • InvalidStateError: wird geworfen, wenn entweder noch nicht das Ziel via XMLHttpRequest.open definiert wurde oder wenn die Anfrage schon via XMLHttpRequest.send gestartet wurde
  • SyntaxError: wird geworfen, wenn ein invalides Header-Feld oder der dazu angegebene Wert invalid ist

[Bearbeiten] XMLHttpRequest.send 

void function send([data=null])

XMLHttpRequest.send schickt die Anfrage los.

Parameter:

  • data: [optional] Daten, die gesendet werden sollen.

Rückgabewert: keine Rückgabe

Ausnahmefehler:

  • InvalidStateError: wird geworfen, wenn entweder die Anfrage nocht nicht via XMLHttpRequest.open geöffnet wurde oder die die Anfrage schon via XMLHttpRequest.send gestartet wurde.

[Bearbeiten] XMLHttpRequest.abort 

void function abort()

Stoppt sämtliche Netzwerkaktivitäten.

Parameter: keine Parameter

Rückgabewert: keine Rückgabe

Ausnahmefehler: keine

[Bearbeiten] XMLHttpRequest.getResponseHeader() 

string function getResponseHeader(string header)

Gibt den Wert eines Antwortheaders zurück, wenn das Headerfeld nicht Set-Cookie oder Set-Cookie2 ist.

Parameter:

  • header: Header-Feldname, groß- und kleinschreibungsunabhängig

Rückgabewert: Wenn ein Header-Feld gefunden wird, wird der Wert zurückgegeben, ansonsten null

Ausnahmefehler: keine

[Bearbeiten] XMLHttpRequest.getAllResponseHeaders() 

string function getAllResponseHeaders()

Gibt Antwortheader zurück (außer Set-Cookie oder Set-Cookie2)

Parameter: keine Parameter

Rückgabewert: Für jedes Headerfeld eine Zeile nach dem Schema <Name>: <Wert>

Ausnahmefehler: keine

[Bearbeiten] XMLHttpRequest.overrideMimeType() 

void function overrideMimeType(string mime)

Überschreibt den MIME-Type des Antwort-Headers

Parameter:

  • mime: stellt den neuen MIME-Type dar

Rückgabewert: keine Rückgabe

Ausnahmefehler:

  • InvalidStateError: wird geworfen, wenn der die Anfrage entweder schon fertig ist, oder gerade geladen wird.
  • SyntaxError: wird geworfen, wenn mime kein gülter MIME-Type ist

[Bearbeiten] XMLHttpRequest.timeout 

int timeout

Beinhalted die Timeout-Zeit für asynchrone Anfragen in Millisekunden. Nach Ablauf der Zeit wird Timeout-Event geschickt.

Ausnahmefehler:

  • InvalidAccessError: wird geworfen, wenn die Anfrage synchron abgearbeitet wird.
  • TimeoutError: wird für XMLHttpRequest.send geworfen

[Bearbeiten] XMLHttpRequest.upload 

XMLHttpRequestUpload upload

Stellt das zur Anfrage dazugehörende XMLHttpRequestUpload-Objekt dar.

[Bearbeiten] XMLHttpRequest.status 

int status

Beinhaltet den HTTP-Statuscode, wenn dieser nicht gegeben ist (aufgrund eines Fehlers oder weil die Anfrage noch nicht gesendet wurde) beinhaltet status den Wert 0.

[Bearbeiten] XMLHttpRequest.statusText 

string statusText

Beinhaltet den HTTP-Statustext, wenn dieser nicht gegeben ist (aufgrund eines Fehlers oder weil die Anfrage noch nicht gesendet wurde) beinhaltet statusText einen leeren String.

[Bearbeiten] XMLHttpRequest.responseType 

string responseType

Beinhaltet den Typ der Antwort, kann verändert werden um den Typ zu ändern.

Mögliche Werte:

  • "" (leerer String)
  • "arraybuffer"
  • "blob"
  • "document"
  • "json"
  • "text"

Ausnahmefehler:

  • InvalidStateError: wird geworfen, wenn responseType gesetzt wird, wenn die Anfrage geladen oder fertig ist
  • InvalidAccessError: wird geworfen, wenn die Anfrage synchron abgearbeitet wird.
