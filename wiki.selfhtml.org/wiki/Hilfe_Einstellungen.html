Die Einstellungsseite ermöglicht angemeldeten Benutzern, bestimmte Teile der Mediawiki-Software an ihre Wünsche und Bedürfnisse anzupassen. Diese Seite erläutert verschiedene Einstellungen.

[Bearbeiten] Benutzerdaten

Allgemeine Einstellungen, die den Benutzereintrag betreffen.

E-Mail-Adresse (optional) 
Geben Sie dort eine E-Mail-Adresse an, wenn Sie mit anderen Benutzern über ein Formular E-Mails austauschen (Siehe: Hilfe:E-Mail).
Unterschrift
Signieren Sie Diskussionsbeiträge mit --~~~~, dann werden die vier Tilden durch Ihren Benutzernamen mit einem Link auf Ihre Benutzerseite ersetzt (--[[Benutzer:Benutzername|Benutzername]]). Wenn gewollt, kann hier eine vom Benutzernamen abweichende Unterschrift angeben werden.
Signatur als Wikitext behandeln (ohne automatische Verlinkung) 
Ist das Kästchen nicht aktiviert, wird die eingestellte Signatur automatisch auf Ihre Benutzerseite verlinkt. Aktivieren Sie dieses Kästchen, wird dieser Link nicht hinzugefügt.
Sprache der Benutzeroberfläche 
Hier kann man die Sprache der Menüelemente der Seiten einstellen.
Passwort ändern 
Hier können Sie Ihr aktuelles Passwort durch ein neues ersetzen.
Altes Passwort 
Geben Sie hier Ihr altes Passwort ein, wenn es geändert werden soll.
Neues Passwort 
Geben Sie hier Ihr neues Passwort ein.
Neues Passwort (nochmal) 
Wiederholen Sie hier Ihr neues Passwort.
Benutzer auf diesem Computer dauerhaft anmelden
(nur beim Anmelden verfügbar) Hier können Sie festlegen, ob die Anmeldung beim SELFHTM-Wiki auf Ihrem Computer mit einem Cookie gespeichert werden soll. Wenn diese Option aktiviert ist, müssen Sie sich nicht jedes Mal, wenn Sie Ihren Webbrowser starten, anmelden, um vom SELFHTML-Wiki erkannt zu werden.
Vermeiden Sie diese Einstellung, wenn Sie sich den Computer mit anderen teilen und über kein eigenes Benutzerkonto verfügen. Sie sollten sich in diesem Fall beim Verlassen des SELFHTML-Wiki zudem immer abmelden.
E-Mail 
Haben Sie eine E-Mail-Adresse angegeben, stehen Ihnen die nachfolgenden Funktionen zur Verfügung:
Bei Änderungen an beobachteten Seiten E-Mails senden 
Sie erhalten jedes Mal, wenn eine von Ihnen beobachtete Seite verändert wird, eine Email. Das gilt nicht, wenn der Autor seine Änderung als "klein" eingestuft hat.
Bei Änderungen an meiner Benutzer-Diskussionsseite E-Mails senden 
Sie erhalten jedes Mal, wenn Ihre Benutzer-Diskussionsseite verändert wird, eine Email.
Auch bei kleinen Änderungen an beobachteten Seiten E-Mails senden 
Sie erhalten jedes Mal, wenn eine von Ihnen beobachtete Seite verändert wird, eine Email. Das gilt auch dann, wenn der Autor seine Änderung als "klein" eingestuft hat.
E-Mail-Empfang von anderen Benutzern ermöglichen 
Diese Option sollte aktiviert werden, wenn man eine E-Mail-Adresse angegeben hat und E-Mails von anderen Benutzern erhalten möchte.

[Bearbeiten] Skin

Es stehen mehrere Skins (Oberflächengestaltungen) zur Verfügung, von denen SELFHTML die Voreinstellung ist. Die Seite Ansicht personalisieren zeigt Ihnen, wie Sie einen Skin an Ihre Wünsche anpassen könnne.

[Bearbeiten] Dateien

Maximale Bildgröße auf Bildbeschreibungsseiten 
Hier kann man Obergrenzen für die Bildgröße angeben, die auf den Bildbeschreibungsseiten nicht überschritten werden sollen.
Größe der Vorschaubilder (Thumbnails) 
Hier gibst man die Größe an, in der die Vorschaubilder angezeigt werden, für die keine spezielle Größe angegeben wurde.

[Bearbeiten] Datum und Zeit

Datumsformat 
Hier wählt man das Datumsformat aus einer Liste von Beispielen aus, in dem Datumsangaben des Systems dargestellt werden sollen, beispielsweise in der Versionsgeschichte von Artikeln.
Zeitzone 
Hier erfolgt die Einstellung der Zeitzone, damit die Software die Zeitangaben subjektiv richtig darstellen kann. Als Hilfe ist die aktuelle Serverzeit angegeben. Die richtige Differenz ist in Deutschland für MEZ im Winter 1:00 und für MESZ im Sommer 2:00. Man kann sie auch Vom Browser übernehmen lassen, wobei dann aber die Sommerzeitumstellung nicht berücksichtigt wird.

[Bearbeiten] Bearbeiten

Hier wird festgelegt, wie man die Bearbeitungsseite aufruft und wie die Seite zum Ändern und Anlegen eines Artikels erscheinen soll.

Zeilen und Spalten 
Hier gibt man die Größe des Text-Eingabefelds an (Standard sind 25 Zeilen und 80 Spalten).
Links zum Bearbeiten einzelner Absätze anzeigen 
Hier wählt man aus, ob für einzelne Absätze Bearbeitungslinks angezeigt werden oder nicht.
Einzelne Absätze per Rechtsklick bearbeiten (benötigt Javascript) 
Bei Rechtsklick auf eine Seite innerhalb eines Abschnitts wird dieser zum Bearbeiten gewählt. Erfordert Javascript (Standard: nein)
Seiten mit Doppelklick bearbeiten (benötigt JavaScript) 
Bei Doppelklick auf eine Seite wird „Bearbeiten“ gewählt. Erfordert Javascript (Standard: nein)
Eingabefeld erweitern, um den ganzen Bildschirm auszufüllen 
Soll das Text-Eingabefeld im Browser immer so breit wie möglich erscheinen? Dies ist im SELFHTML-Skin immer der Fall.
Bearbeiten-Werkzeugleiste anzeigen 
Schaltet die Standard-Werkzeugleiste über dem Text-Eingabefeld ein oder aus. Diese Werkzeugleiste wird für Webbrowser, mit denen sie nicht funktioniert, automatisch ausgeblendet.
Beim ersten Bearbeiten immer die Vorschau anzeigen 
Zeigt eine Vorschau des Artikels schon an, wenn „Bearbeiten“ gewählt wird.
Beachten Sie: Diese Funktion ist fehlerhaft und verschluckt Markup-Formatierungen in der Vorschau.
Vorschau oberhalb des Editierfensters anzeigen 
Soll die Seitenvorschau beim Editieren über oder unter dem Text-Eingabefeld angezeigt werden?
Eigene Änderungen standardmäßig als geringfügig markieren 
Zu aktivieren, wenn gewünscht wird, dass eigene Bearbeitungen automatisch als geringfügige Änderungen („Nur Kleinigkeiten wurden verändert“) markiert werden. Das Anwählen der Option sorgt also dafür, dass standardmäßig das entsprechende Häkchen gesetzt ist.
Externen Editor als Standard benutzen (nur für Experten, es müssen spezielle Einstellungen auf dem eigenen Computer vorgenommen werden) 
…
Externes Programm für Versionsunterschiede als Standard benutzen (nur für Experten, es müssen spezielle Einstellungen auf dem eigenen Computer vorgenommen werden) 
…
Warnen, wenn beim Speichern die Zusammenfassung fehlt 
Falls in die Zeile Zusammenfassung unter dem Editfenster nichts eingetragen wurde, erhält man eine Warnmeldung.

[Bearbeiten] Letzte Änderungen

Hier geht es um das Erscheinungsbild der Seite „Letzte Änderungen“ und einige verwandte Darstellungen.

Anzahl der Tage, die die Liste der „Letzten Änderungen“ standardmäßig umfassen soll 
Hier stellt man die Anzahl der Einträge ein, die auf Seiten mit automatisch generierten Listen angezeigt werden. Dies betrifft nicht nur die „letzten Änderungen“, sondern alle anderen Seiten dieser Art, wie die Versionsübersicht eines Artikels oder Benutzerbeiträge.
Anzahl der Einträge in der Liste der letzten Änderungen, der Versionsgeschichte und den Logbüchern 
dito, nur Anzahl statt Tage.
Kleine Änderungen ausblenden 
Hier kann man wählen, ob man in den „letzten Änderungen“ die als „Kleinigkeiten“ markieren Änderungen sehen will oder nicht.
Kontrollierte Änderungen in den „Letzten Änderungen“ ausblenden 
Die Möglichkeit, Seiten als „kontrolliert“ zu kennzeichnen, wird derzeit nicht verwendet.
Kontrollierte Seiten auf der Liste „Neue Seiten“ verbergen 
Die Möglichkeit, Seiten als „kontrolliert“ zu kennzeichnen, wird derzeit nicht verwendet.
Erweiterte Darstellung der „Letzten Änderungen“ (benötigt JavaScript) 
Wird diese Option gewählt, werden Artikel auf der Seite „Letzte Änderungen“ zusammengefasst.

[Bearbeiten] Beobachtungsliste

In diesem Bereich kannt man einstellen, wie die eigene „Beobachtungsliste“ dargestellt wird. Diese enthält alle Artikel, die man als zu „beobachten“ markiert hat und weist auf aktuelle Änderungen daran hin.

Anzahl der Tage, die auf der Beobachtungsliste standardmäßig angezeigt werden sollen 
Auswahl, für wie viele Tage inklusive heute Änderungen an beobachteten Artikeln dargestellt werden sollen.
Erweiterte Beobachtungsliste zur Anzeige aller Änderungen 
Hier kann eingestellt werden, wie viele Einträge dargestellt werden sollen.
Kleine Bearbeitungen in der Beobachtungsliste ausblenden 
Betrifft die Änderungen, die der Autor mit Nur Kleinigkeiten wurden verändert gekennzeichnet hat.
Bearbeitungen durch Bots in der Beobachtungsliste ausblenden 
Betrifft Änderungen, die von selbständig arbeitenden Programmen durchgeführt wurden.
Eigene Bearbeitungen in der Beobachtungsliste ausblenden 
Ankreuzen, falls man auf der eigenen Beobachtungsliste nur fremde Änderungen sehen möchte.
Bearbeitungen anonymer Benutzer (IP-Adressen) in der Beobachtungsliste ausblenden 
Nicht relevant. Seiten können nur von angemeldeten Benutzern geändert werden.
Bearbeitungen angemeldeter Benutzer in der Beobachtungsliste ausblenden 
Dann sieht man in der Beobachtungsliste gar nichts mehr.
Kontrollierte Änderungen in der Beobachtungsliste ausblenden 
Die Möglichkeit, Seiten als „kontrolliert“ zu kennzeichnen, wird derzeit nicht verwendet.
Selbst erstellte / geänderte / verschobene / gelöschte Seiten automatisch beobachten 
Wenn aktiviert, werden Seiten, die man selbst neu angelegt oder geändert hat, automatisch in die eigene Beobachtungsliste übernommen.

[Bearbeiten] Suche

Hier stellt man ein, in welchen Namensräumen gesucht werden soll und wie die Suchergebnisse einer Volltextsuche erscheinen sollen. (Die Volltextsuche kann bei hoher Serverlast zeitweilig ausgeschaltet sein.)

Treffer pro Seite 
Die maximale Anzahl der Treffer pro Artikel, die angezeigt werden soll.
Zeilen pro Treffer 
Außer dem Artikelnamen werden die ersten Zeilen mit Treffern angezeigt, wenn die laufende Nummer der Zeile nicht größer als diese Zahl ist. Als Zeilen in diesem Sinne werden Absätze, Leerzeilen und Zeilenumbrüche gezählt.
Zeichen pro Zeile 
Die maximale Anzahl der Zeichen, die als Kontext in einer Zeile mit Treffern angezeigt wird.
In diesen Namensräumen soll standardmäßig gesucht werden 
Die Namensräume, die beim Suchen voreingestellt sind.

[Bearbeiten] Verschiedenes

Hier finden sich alle Einstellungen, die nicht in eine der anderen Kategorien passen.

Linkformatierung kleiner Seiten (in Byte) 
Hier kannst du die Obergrenze für die Größe der Artikel bestimmen, zu denen Links farblich besonders markiert werden. Dies ist hilfreich um Links auf besonders kurze Artikel und Begriffsklärungen zu finden. Probiere einen Wert von etwa 300.
Links unterstreichen 
Legt fest, ob Links unterstrichen dargestellt werden sollen oder nicht (Standard: immer)
Links auf nicht vorhandene Seiten hervorheben 
Wenn diese Option gesetzt ist, werden Links auf nichtexistente Seiten (und gegebenenfalls kurze Seiten, siehe oben) farblich hervorgehoben. Andernfalls werden die Links wie normaler Text dargestellt, aber durch ein nachgestelltes ? beziehungsweise ! gekennzeichnet (Standard: ja)
Text als Blocksatz 
Formatiert Fließtext im Blocksatz (Standard: nein)
Überschriften automatisch nummerieren 
In Artikeln mit mehreren Überschriften werden diese mit Nummern dargestellt (Standard: nein)
Anzeigen eines Inhaltsverzeichnisses bei Seiten mit mehr als drei Überschriften 
Zeigt in Artikeln mit mehr als drei Abschnitten ein Inhaltsverzeichnis an (Standard: ja)
auf diesem Computer dauerhaft angemeldet bleiben 
…
Seitencache deaktivieren 
Schaltet das Zwischenspeichern von Seiten aus, falls Probleme mit dem Browser auftreten und neue Seiten nicht entsprechend dargestellt werden (Standard: nein)
„Wechseln-zu“-Links ermöglichen 
Aktiviert zusätzliche Links oben auf der Seite, die für eine automatisierte Sprachausgabe („Screen Reader“) und PDAs gedacht sind; sie sind in einem normalen Browser unsichtbar, auch wenn sie aktiviert sind. Nur sehr alte Browser zeigen diese Links evtl. trotzdem, deshalb kann man sie hier deaktivieren.
Zeige beim Versionsvergleich nur die Unterschiede, nicht die vollständige Seite 
Auswählen, wenn beim Versionsvergleich nur die Unterschiede angezeigt werden sollen.
Zeige versteckte Kategorien 
Es werden auch Kategorien angezeigt, die mittels des Befehls __HIDDENCAT__ in Artikeln ausgeblendet wurden.
Unterschied nach dem Zurücksetzen unterdrücken 
Auswählen, wenn nach dem Zurücksetzen eines Artikels kein Versionsvergleich zu sehen sein soll.
