[Bearbeiten] Variablen in PHP

Variablen-Kapitel im PHP-Handbuch

Man kann in PHP so genannte Variablen verwenden. Diese können verschiedene Werte enthalten und wieder aufgerufen werden.

Variablen werden mit einem Dollar-Zeichen gekennzeichnet ($). Danach folgt der Name der Variablen. Dieser darf nicht mit einer Zahl beginnen.

Die Wertzuweisung erfolgt mit einen Gleichheitszeichen (=). Da es sich um eine Anweisung handelt wird diese mit einem Semikolon abgeschlossen.

Beispiel - Zuweisen eines Strings an eine Variable 

<?php

$text = "Mein Text";

echo $text;

?>

Ausgabe: 

Mein Text

[Bearbeiten] Ausgabekontext beachten

Wann immer Sie den Inhalt von Variablen ausgeben, müssen Sie die Regeln des Systems beachten, in dem der Variableninhalt zu stehen kommt. Es ist dabei unerheblich, ob der Inhalt aus einer vertrauenswürdigen Quelle kommt oder aus einer Benutzereingabe. Eine Nichtbeachtung der Ausgabekontextregeln ist einer der häufigsten Programmierfehler und führt in der Regel zu Injection-Lücken (XSS-Injection, SQL-Injection).

Beispiel: Ausgabe in HTML

Wenn Sie den Variableninhalt in ein HTML-Dokument einfügen wollen, müssen eventuell enthaltene HTML-eigene Zeichen beachtet werden. 

<?php

printf('<input type="text" name="feld" value="%s">', htmlspecialchars($value));

?>

printf() erzeugt ähnlich wie echo und print eine Ausgabe und nimmt zwei oder mehr Paramter entgegen. Der erste ist ein Text mit Platzhaltern. In diesem Fall ist mit %s ein Platzhalter für einen String enthalten. Alle weiteren Parameter stehen für je einen Platzhalter in der angegebenen Reihenfolge. htmlspecialchars() tauscht alle in $value vorkommenden <, >, & und " gegen ihre Entsprechung in HTML aus (&lt;, &gt;, &amp; und &quot;).

Beispiel: Ausgabe in ein SQL-Statment

Um einen String in einem SQL-Statmement zu notieren, müssen vor allem Anführungszeichen und der Backslash beachtet werden. Dabei müssen Sie je nach verwendetem Datenbankmanagementsystem unterschiedlich vorgehen. Das folgende Beispiel bezieht sich auf MySQL und die mysql-Extension von PHP. 

<?php
$sql = sprintf("SELECT feldliste FROM tabelle WHERE feld1='%s' AND feld2='%s'",
               mysql_real_escape_string($variable1),
               mysql_real_escape_string($variable2));
?>

Ausführliche Informationen zu diesem Thema können Sie unter Artikel:Kontextwechsel nachlesen.

[Bearbeiten] Debugging

In einer Variable können beliebige Inhalte stehen und nicht immer ist zur Laufzeit klar, was konkret enthalten ist. Deshalb ist es wichtig, sich beispielsweise im Fehlerfall den Inhalt von Variablen anzeigen zu lassen. Das geht im einfachsten Fall mit echo $variable;. Doch echo erzeugt für einige Variableninhalte keine oder eine ungenaue Ausgabe (null und false erzeugen keine Ausgabe, true ergibt eine 1). Um sich den genauen Inhalt inklusive der Angabe des Typs ausgeben zu lassen, können Sie die Funktion var_dump() verwenden. 

<?php
var_dump($variable);
?>

Ausgaben können zum Beispiel so aussehen: 

string(4) "Test"
int(42)
bool(false)

Um den Inhalt einer komplexen Variable (Array, Objekt) auszugeben, benötigt PHP mehrere Zeilen. Damit die Zeilenumbrüche im HTML-Kontext erhalten bleiben und damit die Übersichtlichkeit der Ausgabe, können Sie vorher ein <pre> ausgeben. Alternativ können Sie sich auch im Browser die Quelltextansicht der Seite anschauen.
