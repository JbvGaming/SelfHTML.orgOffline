[Bearbeiten] JavaScript-Frameworks für komplette HTML-Seiten

Die hier vorgestellten Frameworks dienen dazu, den Inhalt von HTML-Seiten festzulegen.

[Bearbeiten] jQuery

jQuery ist ein leichtgewichtiges JavaScript-Toolkit, das eine einfache Schnittstelle für den DOM bieten möchte. Es ist das meistgenutzte Framework, dank der zahlreich verfügbaren Plugins. Auch ist es für JavaScript-Anfänger relativ einfach möglich, für jQuery-Code zu schreiben. Ziel von jQuery ist es, „die Art und Weise, wie JavaScript geschrieben wird, zu verändern“.

[Bearbeiten] jQuery 1.x

Derzeit werden mit 1.x und 2.x zwei unabhängige Versionsstränge gepflegt, die sich hinsichtlich der Browserkompatibilität unterscheiden. Die Versionen 1.x (ab 1.9) und 2.x besitzen eine kompatible API.


[Bearbeiten] jQuery 2.0

jQuery 2.0. wurde gegenüber der Version 1.x grundlegend umstrukturiert und aufgeräumt. Dadurch wurde die Unterstützung für IE6-8 (und der Kompatibilitätsansicht in aktuelleren Versionen), entfernt.

[Bearbeiten] MooTools

Das Ziel von MooTools ist es, natives JavaScript so zu erweitern, dass vielfältige und mächtige JavaScript-Anwendungen geschrieben werden können. Es möchte dabei alle Möglichkeiten dieser prototypischen Skriptsprache ausnutzen. Die Funktionalität betrifft nicht nur DOM-Manipulation, sondern auch Zeichenketten-, Zahlen-, Array-Funktionen sowie viele andere nützliche Hilfsfunktionen zur Ereignisbehandlung, Effekten, Ajax-Requests und weiterem. Es bietet auch Hilfestellung bei Vererbung und Erweiterung von Funktionen, damit redundanter Code bei Änderungen vermieden wird. MooTools will JavaScript so erweitern, „wie es eigentlich sein sollte“. Zielgruppe sind fortgeschrittene JavaScript-Entwickler. MooTools ist vergleichbar zu Prototype.

[Bearbeiten] Ext

Ext beinhaltet Klassen für Ajax, zur DOM-Manipulation und zur Ereignis-Programmierung. Es kann gut für große Projekte eingesetzt werden und bietet vorgefertigte Benutzersteuerelemente. Außerdem bietet das Ext Möglichkeiten zur Kopplung mit anderen Frameworks.

[Bearbeiten] Prototype

Auf prototype findet man ein Framework, welches die DOM-Objekte im Browser prototypisch überlädt und so einen sehr hohen Grad der Objektorientierung erreicht, was jedoch an anderer Stelle zu Problemen führen kann, etwa wenn man vergisst, Objekt-Eigenschaften bei der Iteration mit hasOwnProperty zu prüfen. Prototype ist vergleichbar zu MooTools.

[Bearbeiten] Yahoo! UI Library

Die Yahoo! UI Library (abgekürzt YUI, das UI steht für User Interface) ist eine große Benutzersteuerelemente-Bibliothek. Mit ihr können interaktive Websites erstellt werden.

[Bearbeiten] JavaScript-Frameworks für SVGs

Anders als die bisher vorgestellen Frameworks, dienen diese hier dazu, SVG-Dokumente zu generieren, um sie innerhalb einer HTML-Seite zu platzieren und fernzusteuern. Somit ist es auf einfachem Wege möglich, Inhalte einer HTML-Seite mit Objekten innerhalb von SVGs zu vermischen.

Beachten Sie: Es ist zwar auch möglich, SVG-Dokumente mittels iframe-, embed- oder object-Tag in HTML-Seiten einzubinden, nur lassen sich die Inhalte solcher SVGs mit starken Einschränkungen manipulieren, wenn ein Skript von der HTML-Seite versucht, auf diese Dateien zuzugreifen. Dies liegt daran, dass die meisten Browser sog. Cross-Site-Scripting unterbinden wollen, selbst wenn sich die HTML-Seite und die SVG-Dokumente auf demselben Server befinden.
Beachten Sie: Die hier vorgestellten Frameworks funktionieren nicht mit dem Internet Explorer!

[Bearbeiten] svg.js

Svg.js ist für ein Framework ziemlich minimalistisch gehalten. Wer nur grundlegende Funktionen benötigt (SVGs generieren, animieren, etc.), der kommt mit nur einer JavaScript-Datei aus. Bei komplexeren Aufgaben, wie z.B. dem Importieren vorhandener SVG-Dokumente, stehen weitere Bibliotheken zur Verfügung. Leider ist dieses Framework nur darauf ausgelegt, eine einzige Grafik zu verwalten.

[Bearbeiten] Snap SVG

Wesentlich mächtiger als svg.js, zeigt sich Snap SVG, welches von der Firma Adobe entwickelt wurde. Die Syntax dieses Frameworks ist ziemlich ähnlich gegenüber svg.js, erlaubt jedoch unter Anderem auch die Zusammenarbeit mit AJAX.
